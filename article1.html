    • Union-Find 4つのアルゴリズム

    1. Quick-Find

    Quick-Findアルゴリズムは、名前が示すように、「Find」操作が非常に高速。 各要素は自身が所属する集合を示すIDを保持しており、同じ集合に所属する要素はすべて同じIDを持つ。 これにより、「Find」操作はO(1)の時間計算量で行うことができる。

    ただし、「Union」操作は遅い。 というのも、2つの集合を併合する際には、一方の集合のすべての要素のIDを更新する必要があるから。 したがって、「Union」操作はO(n)の時間計算量を必要とする。 空間計算量はO(n)。

    Quick_Find.java

    
            public class QuickFind {
                private int[] id;
            
                public QuickFind(int size) {
                    id = new int[size];
                    for (int i = 0; i < size; i++) {
                        id[i] = i;
                    }
                }
            
                public boolean connected(int p, int q) {
                    return id[p] == id[q];
                }
            
                public void union(int p, int q) {
                    int pid = id[p];
                    int qid = id[q];
                    for (int i = 0; i < id.length; i++) {
                        if (id[i] == pid) {
                            id[i] = qid;
                        }
                    }
                }
            }

    2. Quick-Union

    Quick-Unionアルゴリズムは「Union」操作が高速。 「Find」操作はO(n)の時間計算量を必要とするが、「Union」操作は高々O(n)。 空間計算量はO(n)。

    Quick_Union.java

    
            public class QuickUnion {
                private int[] parent;
            
                public QuickUnion(int size) {
                    parent = new int[size];
                    for (int i = 0; i < size; i++) {
                        parent[i] = i;
                    }
                }
            
                private int root(int i) {
                    while (i != parent[i]) {
                        i = parent[i];
                    }
                    return i;
                }
            
                public boolean connected(int p, int q) {
                    return root(p) == root(q);
                }
            
                public void union(int p, int q) {
                    int i = root(p);
                    int j = root(q);
                    parent[i] = j;
                }
            }

    3. Weighted Quick-Union

    Weighted Quick-UnionアルゴリズムはQuick-Unionアルゴリズムを改良したもので、 「Union」操作がバランス良く行われるようになってる。 これにより、「Find」操作と「Union」操作は平均的にはO(log n)の時間計算量で行うことができる。 空間計算量はO(n)。

    Weighted_Quick_Union.java

    
            public class WeightedQuickUnion {
                private int[] parent;
                private int[] size;
            
                public WeightedQuickUnion(int n) {
                    parent = new int[n];
                    size = new int[n];
                    for (int i = 0; i < n; i++) {
                        parent[i] = i;
                        size[i] = 1;
                    }
                }
            
                private int root(int i) {
                    while (i != parent[i]) {
                        i = parent[i];
                    }
                    return i;
                }
            
                public boolean connected(int p, int q) {
                    return root(p) == root(q);
                }
            
                public void union(int p, int q) {
                    int i = root(p);
                    int j = root(q);
                    if (i == j) return;
                    if (size[i] < size[j]) {
                        parent[i] = j;
                        size[j] += size[i];
                    } else {
                        parent[j] = i;
                        size[i] += size[j];
                    }
                }
            }

    4. Path Compression

    Path Compressionは、「Find」操作をさらに高速化するためのテクニック。 具体的には、根へのパスを辿るときに、そのパス上のすべてのノードが直接根を指すようにすることで、 次回以降の「Find」操作の高速化を図る。 これにより、「Find」操作と「Union」操作のアムルト平均時間計算量はほぼO(1)となる。 空間計算量はO(n)。

    以下に、Path Compressionを適用したWeighted Quick-UnionのJavaコードを示す。

    PathCompression.java

    
            public class WeightedQuickUnionWithPathCompression {
                private int[] parent;
                private int[] size;
            
                public WeightedQuickUnionWithPathCompression(int n) {
                    parent = new int[n];
                    size = new int[n];
                    for (int i = 0; i < n; i++) {
                        parent[i] = i;
                        size[i] = 1;
                    }
                }
            
                private int root(int i) {
                    while (i != parent[i]) {
                        parent[i] = parent[parent[i]]; // Path Compression here
                        i = parent[i];
                    }
                    return i;
                }
            
                public boolean connected(int p, int q) {
                    return root(p) == root(q);
                }
            
                public void union(int p, int q) {
                    int i = root(p);
                    int j = root(q);
                    if (i == j) return;
                    if (size[i] < size[j]) {
                        parent[i] = j;
                        size[j] += size[i];
                    } else {
                        parent[j] = i;
                        size[i] += size[j];
                    }
                }
            }

    以上が、Quick-Find、Quick-Union、Weighted Quick-Union、 およびPath Compressionについての説明とそのJavaによる実装例。 それぞれのアルゴリズムはその特性により、適用するべき状況が異なる。 ただし、Path Compressionを適用したWeighted Quick-Unionは最もバランスの良い性能を持つため、 一般的にはこれが推奨される。
